# Obsidian Vault 维护指南

## 目录结构

```
obsidian/
├── 00-入口/           → 导航层（首页、索引、路线）
├── 01-快速开始/       → 入门内容
├── 02-理论基础/       → 理论知识（Transformer、Agent 定义等）
├── 03-技术栈/         → 技术深度文章 ← 新知识加这里
├── 04-实践指南/       → 动手项目、工程实践
├── 05-面试与求职/     → 面试题库 ← 新面试题加这里
│   └── 18-agent-interview-playbooks/  → 深度面试专题
├── 06-路线图/         → 学习路径规划
├── 07-项目与示例/     → 代码模板和项目 README
├── 08-资源/           → PDF、论文、工具链接
├── 09-外部精选/       → 第三方项目（ai-research-ebook 等）
└── 99-归档/           → 过时或废弃内容
```

## 添加新知识文章

### 步骤

1. 确定放置目录（通常是 `03-技术栈/`）
2. 查看当前最大编号，新文件用 `最大编号+1`
3. 创建文件，遵循模板
4. 更新 `00-入口/内容索引.md`

### 文件命名规范

```
{编号}-{英文slug}.md

示例：
30-prompt-engineering.md
31-tool-use-patterns.md
```

### 文章模板

```markdown
# 标题

> 一句话摘要（会显示在 Obsidian 预览中）

## 核心概念

...

## 实践要点

...

## 常见误区

...

## 相关链接

- [相关文章](相对路径.md)
```

## 添加新面试题

### 方式一：追加到现有题库（推荐）

直接在 `05-面试与求职/01~17` 对应文件末尾追加，保持格式一致：

```markdown
#### Q{编号}：问题内容

**难度**：⭐⭐
**公司来源**：字节/阿里/...

**参考思路**：
...
```

### 方式二：新建面试 Playbook（深度专题）

放在 `05-面试与求职/18-agent-interview-playbooks/` 下：

```
{topic}-playbook.md

示例：
multimodal-agent-playbook.md
agent-orchestration-playbook.md
```

### 方式三：新建独立面试题文件

放在 `05-面试与求职/` 下，编号从当前最大值 +1 开始（当前最大：21）：

```
22-{topic}.md
```

## 链接规范

### 相对路径

- 同目录：`[标题](文件名.md)`
- 上级目录：`[标题](../目录/文件名.md)`
- 不要用 `./` 前缀（Obsidian 不需要）

### 文件名含空格

用 `%20` 编码：`[标题](Agent%20Memory%20核心论文汇总.md)`

### 外部链接

直接用完整 URL：`[标题](https://github.com/...)`

## 新增内容 Checklist

## 学习笔记（10-学习笔记/）

学习理论基础和技术栈时产生的延伸思考、主动提问，放在 `10-学习笔记/` 目录。

### 为什么单独放

- 原文是"知识库"——客观、稳定、可复用
- 笔记是"个人理解"——带时间性、主观性、探索性
- 分开后原文可以持续更新而不影响笔记
- Obsidian 反向链接会自动关联两者

### 命名

```
{日期}-{来源slug}-notes.md

示例：
2026-06-01-transformer-notes.md
2026-06-03-mcp-protocol-notes.md
```

### 笔记模板

```markdown
# {主题} 学习笔记

来源：[原文标题](../02-理论基础/03-transformer.md)
日期：2026-06-01

## 延伸问题

### Q1: 问题
> 回答/思考

## 关键洞察

- 重要发现

## 待深入

- [ ] 还没搞懂的点
```

### 日常流程

1. 打开某篇知识文章开始学习
2. 遇到疑问或想延伸的点，在 `10-学习笔记/` 创建对应笔记
3. 笔记开头用链接指向原文
4. 记录问题和思考
5. 后续回顾时可以把"待深入"的点转化为新的学习任务
